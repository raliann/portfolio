Games

Flappy Bird

A C# Windows Forms game, which is a recreation of the popular game Flappy Bird.

Download application

Users can make the bird jump by pressing the space bar. The goal is to avoid the pipes and get the highest score possible. The highest the score, the faster the bird flies forwards. The game starts when the user presses the long button at the bottom of the screen and ends when the bird collides with a pipe or the ground. The score is displayed at the bottom of the screen.

Building this game has strengthened my understanding of real-time logic, physics simulation, and event-driven programming. I implemented gravity, collision detection, and score tracking from scratch, which required precise control over movement, timing, and user input. Designing smooth gameplay taught me how small mathematical adjustments such as velocity changes and frame updates, directly affect user experience.

From a development perspective, the project improved my ability to structure interactive systems and manage game states such as start, play, and game over. Additionally, I retouched my dinamic graphic creating skills in windows forms applications.

Learn provinces in Bulgaria

A C# Windows Forms application that helps users learn the provinces of Bulgaria while playing a game.

Download application

Users are presented with a map of Bulgaria with all the provinces outlined. On its left they are presented with the name of each province. The goal of the game is to match each province with its name by clicking on the province in the list and dragging it to its location. If a mistake is made, the player can correct it by double clicking the wrong province name to remove it from the map. The game ends when all provinces are matched correctly.The time is displayed at the top left of the screen.

This game deepened my understanding of interactive user interfaces and event-driven programming. The process reinforced my knowledge of connecting data structures (like the list of provinces) with visual elements, and managing state changes in response to user interactions, which is a core concept in building responsive applications. By mapping cities to their correct locations on the map, I practiced spatial reasoning in programming and thought critically about user experience, such as ensuring feedback for correct or incorrect placements. This project strengthened my skills in problem-solving, debugging, and designing software that is both functional and intuitive for learners.

Puzzle

A C# Windows Forms application which challenges players by asking them to match squares in order to create a picture.

Download application

Each time the app is started, the puzzle pieces are shuffled. The goal is to put them back in order to create the original picture. To start the game players need to click the button at the bottom left. To move the pieces players need to click on the piece they want to move and drag it to its desired location. If a mistale is made, it can be corrected by double clicking on the incorrect piece and moving it. The game ends when the player clicks the bottom right button. If the puzzle is solved correctly, the player is shown a congratulating message. Otherwise, they are shown a message saying that the puzzle is not solved correctly.

While developing this game, I strengthened my understanding of dynamic game logic and state management. Implementing the piece shuffle functionality taught me how to randomize elements while preserving the underlying structure needed to reconstruct the original image. I learned how to generate different starting configurations each time the game launches, ensuring variability without breaking the solvability of the puzzle. I also deepened my knowledge of event handling and object interaction within a graphical interface. Managing piece selection, dragging, collision detection, and snapping pieces into the correct positions required precise control over coordinates and conditions. This project improved my algorithmic thinking, especially in tracking correct placements and detecting completion, while also sharpening my focus on user experience and game flow.
